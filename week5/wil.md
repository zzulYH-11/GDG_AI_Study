## 결정 트리 (Decision Tree)

스무고개와 유사한 방식으로 데이터를 학습하는 알고리즘으로, 특정 조건(예: 당도가 2보다 작은가?)을 기준으로 데이터를 끊임없이 분기하여 분류 정확도를 높인다.

### 지니 불순도 (Gini Impurity)

노드 내부에서 서로 다른 클래스(레드 와인과 화이트 와인)가 얼마나 섞여 있는지를 나타내는 지표

- 계산식: `1 - (클래스 1 비율의 제곱 + 클래스 2 비율의 제곱)`

순수하게 한 클래스만 존재할수록 0에 가까워지며, 반반 섞여 있을수록 0.5에 가까워진다. 결정 트리는 자식 노드들의 지니 불순도가 낮아지는(순도가 높아지는) 방향으로 분기를 결정한다.

### 규제 기법: 가지치기 (Pruning)

트리의 최대 깊이를 제한하는 `max_depth` 하이퍼파라미터 조절함으로써 결정 트리의 과대적합을 방지한다. 

### 하이퍼파라미터 튜닝 (Hyperparameter Tuning)

개발자가 직접 설정해야 하는 최적의 하이퍼파라미터 수치를 찾기 위해 컴퓨터가 자동으로 조합을 비교해주는 기법이다.

#### **그리드 서치 (Grid Search)**

개발자가 지정한 하이퍼파라미터 후보들의 모든 조합을 하나씩 테스트하는 방식으로, 꼼꼼하게 모든 조합을 비교하여 최적의 값을 찾을 수 있지만 파라미터와 후보가 많아질수록 시간이 매우 오래 걸린다는 단점이 있다.

#### **랜덤 서치 (Random Search)**

후보 범위 안에서 일부 조합을 랜덤하게 샘플링하여 비교하는 방식으로, 모든 조합을 다 보지 않으므로 연산 속도가 빠르지만 전체 조합을 전수 조사하지 않기 때문에 정확성 측면에서 한계가 있을 수 있다

## 검증 세트와 교차 검증

트레인/테스트 세트로만 분할하던 기존 방식에서는 같은 테스트 세트로 모델 성능을 반복 평가하고 규제를 적용하므로 하이퍼파라미터가 테스트 세트에 맞춤화되어 테스트 세트가 오염되는 문제가 생긴다. 테스트 세트는 절대 오염되어서는 안 되며, 실제 성능 평가 시 딱 한 번만 사용해야 한다.

따라서 데이터를 **훈련 / 검증 / 테스트** 세 가지로 분할하고 성능 확인 및 하이퍼파라미터 조정은 검증 세트에서 진행한다.

근데 데이터에서 검증 세트를 추가로 떼어내면 훈련 데이터의 양이 부족해지는 단점 발생한다.

이를 해결하기 위해 데이터를 N등분하여 번갈아가며 검증 세트로 지정하고, 나머지를 훈련 세트로 사용하는 과정을 반복하는 교차 검증 방법을 쓴다. 각 단계에서 나온 검증 점수들의 평균을 최종 검증 점수로 계산한다.

## 앙상블 학습과 랜덤 포레스트 (Random Forest)

- **앙상블 학습 (Ensemble Learning)**

하나의 모델에만 의존하지 않고, 여러 개의 모델을 함께 사용하여 하나의 예측 결과를 도출하는 방법으로, 데이터가 조금만 바뀌어도 구조가 급변하는 결정 트리의 단점을 보완한다.

- **랜덤 포레스트 (Random Forest)**

무작위로 여러 개의 결정 트리(예: 100개)를 만들고, 각 트리의 예측 결과를 다수결로 투표하여 최종 결과를 결정하는 앙상블 기법이다. 만약 개별 트리들이 모두 비슷하면 앙상블 효과가 떨어지므로 두 가지 방법으로 무작위성을 부여한다.

1. **부트스트랩 샘플 (Bootstrap Sample)**: 훈련 데이터에서 중복을 허용하여 랜덤하게 샘플을 추출해 각 트리마다 서로 다른 데이터셋으로 학습시키는 방법. 이 방식은 중복 선택을 허용하므로, 특정 트리의 훈련에 사용되지 않고 남는 데이터(OOB, Out-Of-Bag 샘플)가 생긴다. 이를 별도의 검증 세트처럼 활용하여 모델을 평가할 수 있다.
2. **특성 랜덤 선택**: 노드를 분할할 때 전체 특성이 아닌, 일부 특성(주로 전체 특성 개수의 제곱근만큼)만 랜덤하게 선택하여 사용한다. 이러면 다양한 조합의 특성을 학습한 트리들을 만들 수 있다.
